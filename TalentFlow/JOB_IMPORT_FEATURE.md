# Функціональність імпорту вакансій з зовнішніх сайтів

## Огляд

Реалізована функціональність для автоматичного імпорту вакансій з популярних платформ пошуку роботи. Користувачі можуть вставити посилання на вакансію з іншого сайту, і система автоматично заповнить форму створення вакансії.

## Підтримувані платформи

- **LinkedIn** - Професійна мережа LinkedIn
- **Djinni** - Українська IT платформа для пошуку роботи
- **Robota.ua** - Українська платформа пошуку роботи
- **Work.ua** - Українська платформа пошуку роботи
- **Indeed** - Міжнародна платформа пошуку роботи
- **Glassdoor** - Платформа пошуку роботи та оглядів компаній

## Як використовувати

### 1. Відкрити форму створення вакансії

Перейдіть на сторінку створення вакансії (`/jobs/create`)

### 2. Натиснути кнопку "Перенести з іншого сайту"

У верхній частині форми є кнопка з іконкою зовнішнього посилання.

### 3. Вставити посилання на вакансію

- Скопіюйте URL вакансії з підтримуваного сайту
- Вставте його в поле вводу
- Система автоматично перевірить валідність та підтримку платформи

### 4. Імпортувати дані

- Натисніть кнопку "Імпортувати вакансію"
- Система проаналізує сторінку та витягне основну інформацію
- Після успішного імпорту натисніть "Використати дані"

### 5. Редагувати та зберегти

- Імпортовані дані автоматично заповнять форму
- Внесіть необхідні зміни
- Збережіть вакансію як звичайно

## Що імпортується

Система автоматично витягує:

- **Назва вакансії** - основний заголовок
- **Опис** - детальний опис посади
- **Локація** - місто та країна
- **Компанія** - назва роботодавця
- **Зарплата** - мінімальна та максимальна сума з валютою
- **Тип роботи** - повна/часткова зайнятість, контракт, стажування
- **Рівень досвіду** - junior, middle, senior
- **Навички** - технічні навички з опису
- **Переваги** - додаткові переваги роботи
- **Вимоги** - вимоги до кандидатів

## Технічні деталі

### Backend API

- **POST** `/api/job-parsing/parse` - парсинг вакансії за URL
- **GET** `/api/job-parsing/platforms` - список підтримуваних платформ
- **GET** `/api/job-parsing/validate-url` - валідація URL

### Файли

**Backend:**
- `src/services/jobParsingService.ts` - основний сервіс парсингу
- `src/controllers/jobParsingController.ts` - контролер API
- `src/routes/jobParsingRoutes.ts` - роути API

**Frontend:**
- `components/UI/ImportJobModal.tsx` - модальне вікно імпорту
- `services/jobParsingService.ts` - фронтенд сервіс
- `app/jobs/create/page.tsx` - інтеграція в форму створення

## Обробка помилок

### Типові помилки

1. **Непідтримувана платформа** - URL з сайту, який не підтримується
2. **Невалідний URL** - неправильний формат посилання
3. **Помилка парсингу** - не вдалося завантажити або проаналізувати сторінку
4. **Недоступна сторінка** - вакансія видалена або недоступна

### Рекомендації

- Переконайтеся, що посилання веде безпосередньо на сторінку вакансії
- Використовуйте публічні вакансії (не потребують авторизації)
- Якщо імпорт не спрацював, спробуйте скопіювати дані вручну

## Безпека

- Система не зберігає імпортовані URL
- Використовуються стандартні HTTP заголовки для обходу блокувань
- Валідація всіх вхідних даних
- Обмеження на час завантаження сторінок

## Майбутні покращення

- Додавання нових платформ
- Покращення точності парсингу
- Кешування результатів
- Підтримка авторизованих сторінок
- Інтеграція з AI для кращого розуміння контенту

## Приклад використання

```javascript
// Валідація URL
const response = await fetch('/api/job-parsing/validate-url?url=https://www.linkedin.com/jobs/view/123');
const { data } = await response.json();

if (data.supported) {
  // Імпорт вакансії
  const importResponse = await fetch('/api/job-parsing/parse', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({ url: 'https://www.linkedin.com/jobs/view/123' })
  });
  
  const { data: jobData } = await importResponse.json();
  // Використання jobData для заповнення форми
}
```

## Підтримка

При виникненні проблем з імпортом вакансій:

1. Перевірте, чи підтримується платформа
2. Переконайтеся, що URL валідний
3. Спробуйте інший браузер або очистіть кеш
4. Зверніться до підтримки з деталями помилки
