# Calendar Components

Цей модуль містить компоненти для роботи з календарем подій в TalentFlow.

## Компоненти

### CalendarPage
Основна сторінка календаря з місячним переглядом, навігацією та управлінням подіями.

**Функціональність:**
- Перегляд календаря по місяцях
- Навігація між місяцями
- Створення нових подій
- Редагування існуючих подій
- Відображення подій на календарі
- Бічна панель з подіями на сьогодні
- Позначення подій як завершених

### EventModal
Модальне вікно для створення та редагування подій.

**Функціональність:**
- Форма створення/редагування подій
- Валідація даних
- Підтримка різних типів подій
- Налаштування пріоритетів
- Додавання учасників та тегів
- Налаштування локації та нагадувань

## API Endpoints

### Події
- `POST /events` - Створення нової події
- `GET /events/:id` - Отримання події за ID
- `PUT /events/:id` - Оновлення події
- `DELETE /events/:id` - Видалення події
- `GET /events/month/:year/:month` - Отримання подій за місяць
- `GET /events/week/:year/:week` - Отримання подій за тиждень
- `GET /events/day/:date` - Отримання подій за день
- `GET /events/range` - Отримання подій за діапазон дат
- `GET /events/upcoming` - Отримання майбутніх подій
- `PATCH /events/:id/complete` - Позначення події як завершеної
- `GET /events/stats` - Статистика подій
- `GET /events/search` - Пошук подій

## Типи подій

- `interview` - Інтерв'ю
- `meeting` - Зустріч
- `deadline` - Дедлайн
- `reminder` - Нагадування
- `other` - Інше

## Пріоритети

- `low` - Низький
- `medium` - Середній
- `high` - Високий
- `urgent` - Терміновий

## Використання

### Локальна розробка
1. Запустіть міграцію для створення таблиці подій:
```bash
cd backend
npm run typeorm migration:run
```

2. Запустіть бекенд та фронтенд:
```bash
# Backend
cd backend
npm run dev

# Frontend
cd web
npm run dev
```

### Railway (Production)
1. Запустіть скрипт перестворення бази даних:
```bash
./recreate-database-railway.sh
```

2. Розгорніть зміни:
```bash
railway up
```

## Стилі

Всі стилі для календаря знаходяться в `src/styles/calendar.css` та використовують CSS змінні дизайн-системи.

## Локалізація

Календар підтримує 3 мови:
- Українська (uk)
- Російська (ru)
- Англійська (en)

Всі тексти використовують функцію `t()` з контексту `useLanguage`.

## Залежності

- `lucide-react` - Іконки
- `date-fns` - Робота з датами (опціонально)
- TypeORM - База даних
- React Context - Управління станом
